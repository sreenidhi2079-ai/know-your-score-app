# 🧭 STEP 3 — Feature Engineering & Label Creation

## 3.1 Feature Selection

Selected key behavioral features affecting smartphone usage:

* Avg Screen Time (hours per day)
* Phone Checks Per Day
* Hours Without Phone

---

## 3.2 Addiction Score Creation

* Created a scoring system based on selected features
* Assigned weights based on usage intensity:

  * Higher screen time → higher score
  * More phone checks → higher score
  * Fewer hours without phone → higher score

---

## 3.3 Addiction Level Classification

Converted score into categorical labels:

* **Low** → Minimal smartphone dependency
* **Medium** → Moderate usage behavior
* **High** → Strong dependency / addictive behavior

# 📄 STEP 4 — Machine Learning Model Training

---

## 🧭 4.1 Feature Selection

* Selected key behavioral features from the dataset:

  * Avg_screen_time_hours
  * Phone_checks_per_day
  * Hours_without_phone

* These features represent smartphone usage intensity and dependency behavior.

---

## 🧭 4.2 Target Variable Preparation

* Defined target variable:

  * **Addiction_Level (Low / Medium / High)**

* Converted categorical labels into numerical format using **Label Encoding**:

  * Low → 0
  * Medium → 1
  * High → 2

---

## 🧭 4.3 Train-Test Split

* Dataset was split into:

  * **Training set (80%)** → used to train the model
  * **Testing set (20%)** → used to evaluate performance

* This ensures model is tested on unseen data for unbiased evaluation.

---

## 🧭 4.4 Model Training

* Machine Learning Algorithm used:

  * **Decision Tree Classifier**

* Model was trained using training data:

  * Learns patterns between smartphone usage behavior and addiction level

---

## 🧭 4.5 Model Evaluation

* Model performance evaluated using **Accuracy Score**

* Achieved accuracy:

  * **~90%**

* Indicates the model is performing well on unseen data.

---

## 🧭 4.6 Model Saving

* Trained model saved using **Joblib**:

  * model.pkl → trained ML model
  * label_encoder.pkl → label encoder for output decoding

* This allows model to be reused in web application without retraining.
